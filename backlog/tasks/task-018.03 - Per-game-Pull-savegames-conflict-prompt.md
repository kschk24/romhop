## Agent Brief

**Category:** enhancement
**Summary:** Add 'Pull savegames' as a per-game action that pulls one game's saves + savestates from RomM, prompting per conflict, with a written/skipped/kept/failed summary. Depends on TASK-018.02.

**Current behavior:**
After TASK-018.02, the action dispatch router has a `"pull"` branch that is a no-op (guarded stub). There is no per-game pull UI path.

**Desired behavior:**
The 'Pull savegames' action (context menu + panel button) pulls saves and savestates for one game from RomM. Pull works regardless of whether the game is downloaded (saves land in saves/states dirs). Runs off the UI thread. When a local file differs from RomM's copy, a dialog prompts per conflict showing both timestamps (remote `updated_at` vs local mtime); user picks keep-local or take-remote. A summary (written/skipped/kept/failed) is shown on completion.

**Key interfaces / contracts:**
- The existing `pull_games(client, entries, settings, *, on_conflict=None, on_written=None, on_error=None) -> dict` is reused. Pass a one-element iterable of a small shim exposing `.rom_id = rom.id` — no `mapping_cache` dependency.
- `PullWorker(pull_fn)` — new QThread worker; `pull_fn(on_conflict) -> dict` runs pull off the thread. On conflict: emits `conflict(item, local_path, local_mtime)` signal and blocks on a `threading.Event` until the UI thread calls `resolve_conflict(take_remote: bool)`. Emits `done(summary)` or `failed(message)`.
- `PullConflictDialog` — modal dialog; shows file name, remote `updated_at`, local mtime; returns `True` for take-remote, `False` for keep-local. Class method `ask(file_name, remote_updated, local_mtime, parent) -> bool` for fire-and-return use.
- `MainWindow._pull_one(rom)` — replaces the Task-8 stub; creates a `PullWorker`, connects `conflict` → show dialog → `resolve_conflict`, connects `done` → summary `QMessageBox`, connects `failed` → error `QMessageBox`. Emits `pull_finished` signal. `MainWindow` gains `pull_action` injected callable.
- `app.run()` composes `pull_action(rom, on_conflict) -> dict` by calling `pull_games` with the one-rom shim.

**Acceptance criteria:**
- [ ] Pull savegames action pulls saves+states for one game using a one-rom shim over `pull_games` (`.rom_id` only, no mapping_cache)
- [ ] Pull runs off the UI thread and works whether or not the game is downloaded
- [ ] Differing local file prompts per conflict with both timestamps; keep-local and take-remote are both honored
- [ ] Conflict prompt marshals from worker thread to UI thread without freezing or crashing
- [ ] A summary of written/skipped/kept/failed is shown after the pull; tests cover the shim and conflict resolution paths

**Out of scope:**
- Batch pull from the panel (batch ops stay checkbox-driven)
- Conflict resolution for the batch CLI pull path (separate feature)
- Editing or uploading saves to RomM from the panel
