## Implementation Plan

<!-- SECTION:PLAN:BEGIN -->
Root cause: src/romhop/gui/app.py:307 runs `_sys.exit(app.exec())` with no SIGINT handler. Qt's C++ event loop blocks Python signal delivery, so Ctrl-C is never seen; a second Ctrl-C/SIGTERM tears down Qt mid-state -> segfault.

Fix (preferred, clean): before app.exec(), install a SIGINT handler that calls app.quit(), and start a QTimer (~200ms, no-op lambda) so the Python interpreter regains control periodically and the handler can fire. On quit, ensure sync stop_event is set and the tray icon is removed (reuse existing window/sync shutdown path). Avoid bare signal.SIG_DFL: it skips Qt cleanup and would leave the sync thread/tray dangling.

Files: src/romhop/gui/app.py (run()). Likely 1 file + 1 test in tests/ (pytest-qt qtbot). Keep backend Qt-free boundary intact.

## Agent Brief — GUI won't exit on Ctrl-C, segfaults on force

**Confirmed root cause (traced in code):** `src/romhop/gui/app.py:307` ends with `_sys.exit(app.exec())`. No SIGINT handler is installed anywhere (grep clean). Qt's C++ event loop blocks Python signal delivery, so Ctrl-C from the launching terminal is swallowed; forcing it (2nd Ctrl-C / SIGTERM) destroys Qt objects mid-event -> segfault. Matches reporter: "mit ctrl c beenden geht nicht ... seg fault wenn man es forct."

**What to do:** Install a SIGINT handler -> `app.quit()`, plus a periodic QTimer (~200ms no-op) so the interpreter wakes often enough to run the handler while Qt loops. On quit, set the sync `stop_event` and remove the QSystemTrayIcon so nothing is orphaned (background-sync/tray feature already exists — reuse its shutdown).

**Do NOT** just set `signal.SIG_DFL`: abrupt exit skips Qt teardown and leaves the sync thread + tray icon dangling.

**Boundary:** keep backend Qt-free; change stays in `gui/app.py`. Add a pytest-qt test asserting the handler is installed / quit path reachable.
