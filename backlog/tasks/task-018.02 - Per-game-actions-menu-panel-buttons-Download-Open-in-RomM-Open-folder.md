## Agent Brief

**Category:** enhancement
**Summary:** Expose Download, Open in RomM, and Open containing folder as per-game actions in both the tile right-click context menu and the Detail panel buttons. Depends on TASK-018.01.

**Current behavior:**
After TASK-018.01, the Detail panel exists and tiles emit `tile_activated`. There are no per-game action buttons on the panel and no right-click context menu on tiles.

**Desired behavior:**
Right-clicking a tile shows a context menu; the Detail panel shows matching buttons. Both surfaces expose the same four actions (Pull savegames is TASK-018.03): Download, Open in RomM, Open containing folder. All actions injected into `MainWindow` as callables — widgets never import backend modules.

- **Download**: reuses the existing batch download machinery scoped to one rom. Label is 'Re-download' when already downloaded.
- **Open in RomM**: opens `{romm_url}/rom/{id}` in the system browser (`QDesktopServices`). The URL pattern lives as a single module-level constant (`romm_game_url(base_url, rom_id) -> str`).
- **Open containing folder**: opens the game's on-disk folder via `QDesktopServices`. Disabled/greyed out when the game is not downloaded. Folder resolved via a pure helper (`local_game_dir(rom, roms_root, overrides) -> Path | None`) using the same platform-slug→system-dir mapping and ES-DE layout as the downloader.

**Key interfaces / contracts:**
- `LibraryView.action_requested = Signal(str, object)` — emits `(action_name, rom)` where `action_name ∈ {download, pull, open_romm, open_folder}`. Context menu disables 'Open containing folder' when the rom is not in the downloaded set.
- `DetailPanel` gains four action signals: `download_requested(rom)`, `pull_requested(rom)`, `open_romm_requested(rom)`, `open_folder_requested(rom)`.
- `MainWindow._dispatch_action(name: str, rom)` — single router connected to both `LibraryView.action_requested` and each `DetailPanel.*_requested` signal.
- `romm_game_url(base_url: str, rom_id: int) -> str` — pure helper, strips trailing slash, single constant for the path pattern.
- `local_game_dir(rom, roms_root, overrides) -> Path | None` — returns the game's subfolder (multi-disc) or system dir (flat cartridge), or `None` when not present.
- New `MainWindow` injected params: `open_in_romm`, `open_folder`; composed in `app.run()`.

**Acceptance criteria:**
- [ ] Right-click on a tile shows a context menu with Download, Open in RomM, Open folder; the Detail panel shows the same actions as buttons
- [ ] Download triggers a single-game download reusing existing machinery; label reflects downloaded state ('Re-download' if downloaded)
- [ ] Open in RomM opens `{romm_url}/rom/{id}` in the browser via an injected callable; URL pattern is a single constant
- [ ] Open containing folder opens the game's on-disk folder and is disabled when the game is not downloaded
- [ ] Actions injected in `app.run()` (no backend imports in widgets); tests cover menu/button wiring and folder-path resolution

**Out of scope:**
- Pull savegames action (TASK-018.03)
- Batch actions (stay checkbox-driven)
- Editing RomM metadata
