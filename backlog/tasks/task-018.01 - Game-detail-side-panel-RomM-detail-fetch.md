## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Part 1/3 of the game-detail-panel feature (design spec: docs/superpowers/specs/2026-06-17-game-detail-panel-design.md).

Add a Detail panel docked on the right of MainWindow. Clicking a tile's BODY (not the checkbox) opens it; clicking another tile updates it in place; a close button dismisses it. The tile's QCheckBox(name) must split into a bare checkbox glyph (batch-select only) + a separate name label so body-click no longer toggles selection.

Panel shows local Rom fields instantly (cover, name, platform, file list, downloaded status), then fills in RomM-fetched detail when it arrives: summary, release date, genres, file size. Add RommClient.get_rom(id) returning these (all OPTIONAL, degrade gracefully — field names need a live probe of GET /api/roms/{id} on the real server first, MockTransport tests only). Fetch off the UI thread via a worker; cache result per rom id for quick game-to-game flipping. On fetch failure, keep local fields + show a 'couldn't load details' note.

## Agent Brief

**Category:** enhancement
**Summary:** Add a right-docked Detail panel to the library grid, opened by clicking a game tile body, showing local metadata instantly and RomM-fetched detail asynchronously.

**Current behavior:**
Every tile in the library grid contains a QCheckBox with the game name as its label. Clicking anywhere on the tile toggles the batch-select checkbox. There is no way to inspect a single game's metadata or act on it individually.

**Desired behavior:**
Clicking a tile's body (cover image or name label, but NOT the checkbox glyph) opens a Detail panel docked on the right of the main window. The panel shows local fields immediately (name, platform, file list, downloaded status), then fills in RomM-fetched fields when they arrive: summary, release date, genres, file size. The checkbox glyph continues to toggle batch selection without opening the panel. Clicking another tile updates the panel in place. A close button hides it.

**Key interfaces / contracts:**
- `RommClient.get_rom(rom_id: int) -> RomDetail` — new method; `RomDetail` has fields `summary: str | None`, `release_date: str | None`, `genres: list[str]`, `file_size: int | None`. Every field parsed defensively (tolerates any field being absent). **Field names must be confirmed against the live server before committing** — probe `GET /api/roms/{id}` on the live RomM instance and adjust key names (`first_release_date_string`, `fs_size_bytes` are assumed but unverified). Auth token already has `roms.read` scope — no new key needed.
- The tile widget splits: a bare `QCheckBox` (no text, batch-select only) + a separate `QLabel` carrying the name. Body click (cover or name) emits `tile_activated(rom)` signal on `LibraryView`; checkbox continues to emit `selection_changed`.
- `DetailPanel.set_rom(rom)` shows local fields immediately and kicks an off-thread fetch (skip if cached per `rom.id`). Fetch failure keeps local fields + shows 'couldn't load details'. Cache hit shows cached detail instantly.
- `DetailPanel.set_downloaded(ids: set[int])` gates the Open folder button (added in TASK-018.02) and Download/Re-download label.
- `MainWindow` gains a `detail_provider` injected callable; `DetailPanel` is docked right of the content stack; `LibraryView.tile_activated` → `panel.set_rom` + show; `panel.closed` → hide.

**Implementation guide:** Design spec and full TDD plan with exact test + implementation snippets already exist; see task references.
