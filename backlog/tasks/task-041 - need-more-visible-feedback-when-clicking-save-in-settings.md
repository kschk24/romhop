## Agent Brief

**Category:** enhancement
**Summary:** Post a toast when settings are saved so the user gets visible confirmation.

**Current behavior:**
`SettingsView._on_save()` persists config and emits the `saved` signal. `MainWindow._on_settings_saved()` picks it up, updates internal state and the sync toggle тАФ but shows nothing to the user. The settings form stays open with no visual change.

**Desired behavior:**
After saving, a brief info-level toast appears (bottom-right, auto-dismisses) confirming settings were saved. The toast should use the existing `ToastManager` / `ActivityEvent` infrastructure already wired in `MainWindow`.

**Key interfaces:**
- `ActivityKind` enum in `romhop/activity.py` тАФ add a `SETTINGS_SAVED` value (or reuse an existing non-error kind; the toast renders any non-error kind as `ToastInfo`)
- `ActivityEvent` dataclass тАФ construct with the new kind and a short message like `"Settings saved"`
- `MainWindow._on_settings_saved()` тАФ call `self._activity_hub.post(ActivityEvent(...))` here; `_activity_hub` is already connected to `_toast_manager`
- `ActivityEvent.is_error` property тАФ determines toast styling; must be `False` for this event

**Acceptance criteria:**
- [ ] Clicking Save in settings shows a brief toast with a "Settings saved" message
- [ ] Toast renders in info style (not error style)
- [ ] Toast auto-dismisses (existing ToastWidget timeout behavior)
- [ ] No new UI widgets or layout changes introduced
- [ ] Existing `pytest` + `pytest-qt` suite passes

**Out of scope:**
- Changing Save/Cancel button behavior or navigation
- Validation UI or error toasts for bad settings (separate concern)
- Any changes to the toast auto-dismiss timing or styling
