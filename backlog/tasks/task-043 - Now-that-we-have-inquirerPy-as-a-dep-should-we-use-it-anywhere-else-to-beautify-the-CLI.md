## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
Originally a question ("now that InquirerPy is a dep, beautify the CLI elsewhere?"). Grilled down to two pain-free-but-real-value spots, dropping aesthetic repaint.

Two CLI selection points currently dead-end with `exit 2` on ambiguity ("be more specific"). Replace that dead-end with an InquirerPy multi-select when interactive, bring `download` to feature-parity with the GUI's batch download, and let `download` take an explicit list of game names.

Scope:
- `download <name...>`: accepts multiple name args (variadic). Each resolves; exact/single matches go straight into the batch, an ambiguous term opens a checkbox multi-select (tty) scoped to that term. All resolved games download sequentially as a batch, mirroring GUI DownloadWorker.
- `pull <name>`: ambiguous match → checkbox multi-select → pull saves/states for all ticked games.

Out of scope: `config set`, `config set-platform`, `config set-core` (dropped), and any aesthetic-only repaint of `setup`/confirm prompts.
